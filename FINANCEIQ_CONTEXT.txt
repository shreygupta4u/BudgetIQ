## OVERVIEW
FinanceIQ is a Canadian personal budgeting app built with Python + Streamlit.
Backend: SQL Server Express (localhost\SQLExpress, database: FinanceIQ).
No AI used for categorization — rules-based only.
Run: streamlit run app.py

## TECH STACK
- Python 3.8+
- streamlit >= 1.35
- pandas, plotly, streamlit-option-menu, openpyxl, pyodbc
- SQL Server Express (pyodbc connection string in database.py)

## FILES
1. app.py           — Main Streamlit app (all pages / UI)
2. database.py      — All SQL Server DB operations
3. parsers.py       — CSV parsers for 5 Canadian institutions
4. categorizer.py   — Rule-based transaction categorizer
5. Database_Setup.sql — Full DB creation + seeding script (run once)
6. requirements.txt — Python dependencies

## DATABASE SCHEMA  (8 tables)

### parent_categories
  id INT PK, name NVARCHAR(100) UNIQUE, color NVARCHAR(7),
  icon NVARCHAR(10), is_income BIT DEFAULT 0

### child_categories
  id INT PK, parent_id INT FK→parent_categories,
  name NVARCHAR(100), color NVARCHAR(7), icon NVARCHAR(10),
  is_income BIT DEFAULT 0

### accounts
  id INT PK, name, institution, type (checking/credit/investment/mortgage),
  account_number, currency DEFAULT 'CAD', balance DECIMAL(18,2),
  created_at, UNIQUE(institution, account_number)

### transactions
  id, account_id FK, date, description, original_desc, amount DECIMAL(18,2),
  parent_category_id FK, child_category_id FK, memo,
  is_split BIT, is_transfer BIT, transfer_to_account_id FK,
  transfer_investment_cat, cleared BIT, currency,
  tags NVARCHAR(MAX) [JSON array], ai_confidence DECIMAL(5,2),
  hash NVARCHAR(100) UNIQUE [MD5 dedup fingerprint], imported_at

### split_transactions
  id, transaction_id FK→transactions ON DELETE CASCADE,
  child_category_id FK, amount DECIMAL(18,2), memo

### categorization_rules
  id, pattern NVARCHAR(255), child_category_id FK,
  match_type (contains/startswith/exact/regex),
  priority INT, use_count INT, created_at

### budgets
  id, child_category_id FK, month_year DATE (first of month),
  amount DECIMAL(18,2),
  carry_forward BIT DEFAULT 0,
  carry_forward_type NVARCHAR(50) DEFAULT 'none'
    — values: none / underspent / overspent / both
  UNIQUE(child_category_id, month_year)
  Budget lookup: get most-recent row where month_year <= requested month

### goals
  id, name NVARCHAR(255), goal_type (total/monthly),
  target_amount DECIMAL(18,2), monthly_amount DECIMAL(18,2),
  target_date DATE, account_id FK→accounts, created_at

## CATEGORIZATION LOGIC (categorizer.py)
Priority: DB rules → fast regex rules → "Uncategorized"
- DB rules checked first (user-saved, highest priority)
- 100+ Canadian merchant regex patterns compiled at startup
- Manual correction auto-saves a keyword rule (learn_from_correction)
- categorize_batch() used on import

## CSV IMPORT (parsers.py)
Supported institutions: Wealthsimple Cash, Rogers Mastercard, Questrade,
Wealthsimple Invest, CIBC Mortgage
- Auto-detects account name + card/account number
- MD5 hash dedup: account+date+description+amount
- Multiple files can be uploaded at once
- ACCOUNT_TYPES dict maps institution → (institution_name, acc_type)
- INSTITUTION_PARSERS dict maps institution → parse function

## DEDUPLICATION
Every transaction fingerprinted as MD5(account+date+description+amount).
Duplicate hash → silently skipped on import (pyodbc.IntegrityError on UNIQUE constraint).

## TAGS SYSTEM
- tags column in transactions: JSON array string, e.g. '["vacation","tax-deductible"]'
- db.get_all_tags() returns all unique tags
- db.update_transaction(id, tags=[...]) saves tags
- Filter panel: multiselect on tags
- 🏷 button on each transaction row opens tags_dialog
- tags_dialog: shows current tags (click to remove), add from existing, create new

## BUDGET EFFECTIVE DATE LOGIC
- upsert_budget() accepts effective_date param (overrides month_key)
- get_budgets(month_key) returns the MOST RECENT budget row per category where month_year <= requested month
- This means changing a budget in March only affects March+, not Jan/Feb
- carry_forward_type: 'none'=fresh each month, 'underspent'=roll unused, 'overspent'=deduct deficit, 'both'=both

Adds missing columns to existing DB without requiring full rebuild:

- goals table (CREATE IF NOT EXISTS)
- budgets.carry_forward, budgets.carry_forward_type

## KEY DESIGN DECISIONS
1. Category badge in transactions is a colored inline span (not a bare selectbox) —
   selectbox appears for non-split rows, colored box for split rows
2. Split dialog: 2 rows default, first row gets full amount, second gets 0.
   Auto-balance: when row 0 changes, row 1 = total - row0 (triggers st.rerun)
3. Budgets: NO separate Set Budgets tab. All editing via ✏️ inline icon.
4. Report tabs are st.button elements in a row, not stTabs, for custom styling.
5. "Mark as Transfer" button removed from transaction rows.
6. Filter panel is a collapsible section (not inline filters) like Quicken/Monarch Money.
7. Tags stored as JSON array in transactions.tags column.
